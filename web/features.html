Feature page

Everything behind the round.

The homepage sells the energy. This page explains the machine: prompt depth, repeat control, optional AI refill, live health, admin setup, and why the flow stays quick once the room starts moving.

Add to Discord Read Docs
Core stack

Fast rounds with less admin friction.

  • Local truth and dare pools give instant coverage before any AI request is needed.
  • Recent prompt history keeps individual channels from looping the same round energy.
  • Live health reporting exposes the actual bot state instead of hiding behind a static page.
  • Support, docs, and legal now live in their own pages so home stays focused.

Breakdown

What the bot is doing under the hood.

01

Prompt depth first

Large local pools make the bot feel loaded from the first invite instead of acting empty until external services are wired in.

02

Recent-history shield

Repeat control works per room so a busy server can keep playing without hearing the same recycled prompt too soon.

03

Optional AI backup

When enabled, AI behaves like a refill layer, not the whole product. The local pool remains the core experience.

04

Live supervision

The site reads actual runtime state so you can see boot, cooldown, reconnect, and ready signals instead of guessing.

05

Button-first continuation

Rounds stay moving after the initial command through quick actions that keep the room engaged.

06

Safer public fit

Guardrails keep the tone playful and sharp without turning the bot into unusable explicit spam.

For players

The room feels faster.

Users only need a command and a button. The rest of the experience is about pace, freshness, and a clear next move.

  • Low pause between rounds
  • Prompt rotation that feels varied
  • Support server available when something looks off
For admins

The setup feels controllable.

Admin-facing pages, command docs, and live runtime visibility make it easier to invite, test, and keep the bot healthy after launch.

  • Dedicated docs page for setup
  • Direct status path instead of guessing
  • Legal/support pages ready for public launch
