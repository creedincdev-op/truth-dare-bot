Fast answer

If something looks off, check support first.

  • Use the support page for the direct server invite and GitHub link.
  • Use live health if you need to confirm whether the bot is up.
  • Use docs if commands are the only thing you need.
Do I need OpenAI for the bot to work?

No. The bot already has local prompt pools. OpenAI is optional and only acts as a fresh-prompt layer when configured.

How does anti-repeat work?

The bot tracks recent prompt history so the same channel does not immediately recycle the same truths or dares.

Where do I get support?

The support page includes the support server invite and GitHub repo so users have a clear place to go after the homepage.

How do I start a round?

Run /maxplay, then continue with the action buttons to keep the game moving.

Can I see whether the bot is actually online?

Yes. The site reads live runtime state when served through the bot server, so you can see boot, cooldown, or ready state.

Is the website static or live?

It supports both. Local file previews use fallback content, and the served version can fetch live site data and runtime health.

Why are there separate pages now?

To keep the homepage clean. Support, docs, FAQ, and legal pages can hold the detailed stuff without turning home into a dump page.

What if slash commands do not appear?

Re-invite with the correct permissions, wait for Discord propagation, and then verify from the docs/support flow before assuming the bot is broken.

Need an actual route?

Use the support page for the next move.

That page is where the server invite, GitHub link, and health path live now.
